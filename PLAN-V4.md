# PLAN-V4 — сводный план работ по сайту (site-os)

Сведено из 7 экспертных отчётов. 7-й отчёт («test/probe») — мусорный прогон, отброшен целиком.
Дубли слиты, противоречия сняты, всё, что ломает выверенные вещи или жёсткие ограничения брифа, вынесено в раздел «НЕ делаем».

Общий принцип приоритезации внутри лота: сверху — максимум пользы для заявки при минимуме трудозатрат.

## Порядок работ (пересечения по файлам)

Все три лота трогают `os.css`, `os.js`, `index.html`. Критичные зависимости:

1. **ЛОТ 3.4 (resize-хендлер) → перед ЛОТ 1.2/1.4.** Сброс инлайн-геометрии при десктоп↔мобайл живёт в 3.4; мобильная `visibility:hidden` (1.2) и перенос blur (1.4) опираются на корректное состояние закрытых окон.
2. **ЛОТ 2.7 (иерархия заголовков, index.html) согласовать с ЛОТ 1.5 (остров/статус-бар в том же index.html)** — делать в одной ветке, чтобы не конфликтовать по разметке шапки.
3. **ЛОТ 2.4 (`--text-dim`) → перед ЛОТ 2.9 и мелкими 11px-проверками** — токен меняется один раз, потом перепроверяются узлы.
4. **ЛОТ 3.5 (активное окно) и 3.7 (глифы трафиклайтов)** — один блок: серые трафиклайты без символов = следствие неактивного состояния.
5. Иконки (ЛОТ 3.1) не блокируют ничего, но временные — если запланирована замена на свои SVG, сжатие делать уже на финальных иконках, чтобы не пересжимать дважды.

## НЕ делаем (и почему)

- **Аналитика (Яндекс.Метрика и любой счётчик с загрузкой извне)** — прямое нарушение жёсткого ограничения «ноль внешних ресурсов». Риск для измерения конверсии принят осознанно (см. бриф п.4).
- **Форма захвата лида с реальной отправкой** — нет бэкенда, внешний сабмит нарушает то же ограничение. Максимум допустимого позже — mailto-форма без сети; сейчас отложено, прямые контакты остаются.
- **Landscape-раскладка мобильного** (медиазапросы orientation) — низкая польза: аудитория смотрит с телефона в портрете; трудозатраты не окупаются заявкой.
- **Клавиатурная альтернатива drag/resize окон** — весь контент доступен без перемещения (авто-каскад, Esc, кнопка «Назад»); низкий приоритет 2.5.7.
- **Радиус окна 16→10px, «отражение» пола дока** — это осознанный выбор по DNA displace.agency, зафиксирован брифом; PASS-пункты аудита. Не трогаем.
- **Любые параметры магнификации дока** (52→84, R=132, stiffness 190/damping 20) — прямой запрет брифа, выверено по эталону.
- **Правки текстов продукта, H1-формулировок, description, 3× JSON-LD, самого факта hreflang** — кроме 4 согласованных в ЛОТ 2 и SEO-бага canonical (3.8). Плейсхолдер `https://DOMAIN/` оставляем.
- **Полная имитация клавиатурного меню (role=menu со стрелками)** — вместо этого роли меню убираем (2.8): дешевле и честнее для AT.
