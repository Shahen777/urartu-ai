## Смысловой сдвиг (важнее всего)

Было: «ИИ-ассистент по документам». Становится: **«Локальные ИИ-сотрудники, работающие 24/7»**.
Продаём штат цифровых сотрудников, которые живут на сервере клиента: не спят, не болеют, не увольняются, данные не выносят. Ассистент по документам — лишь один из них. Каждый экран сайта должен отвечать на вопрос директора: «какую работу это делает за меня и почём».

Сайт = «компьютер компании, где эти сотрудники уже работают». Десктоп = Mac, мобильный = iPhone. Это уже построено — доводим до ума и наполняем функциональностью.

## ЛОТ A — «Мобильный iPhone без ошибок» (модель: Fable; самый сложный)

### A1. Свайп на реальном iPhone не работает (критический баг)
Синтетические pointer-тесты проходят, но на живом iOS Safari листание вправо/влево не идёт. Почти наверняка: iOS Safari отбирает жест и шлёт `pointercancel`, а `touch-action`/`overscroll-behavior` его не останавливают.
Требуется:
- Переписать пейджер на **touch events с `passive:false`** (touchstart/touchmove/touchend) с ручным определением горизонтали и `preventDefault()` на горизонтальных движениях; pointer events оставить для мыши/пера. Не полагаться на setPointerCapture на тач.
- **Проверить на настоящем WebKit**: установить движок (`npx playwright install webkit` — можно из ~/Projects/the-delegation, там есть node_modules, или глобально) и прогнать реальный тач-тест (`page.touchscreen`/CDP-жесты) на 390×844: свайп влево → страница 2, свайп вправо → страница 1, вертикальный скролл внутри страницы не сломан, тап по иконке всё ещё открывает окно, свайп не срабатывает от простого тапа.
- Проверить, что жест Пункта управления (из правого верхнего угла) и свайп-закрытие листов тоже работают на touch events.

### A2. Обои («фон не нравится»)
Текущие SVG-обои бледные и грязноватые. Нарисовать новые `wallpapers/dark.svg` и `light.svg` уровня macOS Sequoia:
- Светлые: чистая небесно-голубая → мятная → тёплая персиковая волна (как штатные обои iPhone на скриншоте клиента), сочные, но НЕ кислотные; тёмные: глубокий индиго-синий → бирюза → тёплый янтарный отсвет у горизонта, без фиолетового доминирования и без чёрной жижи.
- Техника: крупные перекрывающиеся blob-формы + feGaussianBlur 60–110 + лёгкое зерно (feTurbulence, opacity ≤0.04). Вес каждого файла ≤80KB. 1600×1000.
- Обязательный тест: белые подписи читаются на тёмных, тёмные подписи на светлых — на ЛЮБОЙ области обоев (проверить в 4 точках экрана замером).

### A3. Сквозная читаемость («буквы не читаются»)
Аудит контраста ОБЕИХ тем на мобильном и десктопе: подписи иконок, виджеты (значения и подписи), тексты окон, кнопки, статус-бар, док-подписи, Пункт управления. Всё ниже 4.5:1 — чинить (цвет/подложка/ореол). Замеры в отчёт.

### A4. iPhone-оболочка — добить точность
- Строка «Поиск»: у Apple она серо-полупрозрачная с blur — сверить.
- Виджеты на странице 2: iOS-стиль (радиус 22, крупные значения), сетка 2×N где уместно, а не только столбец.
- Открытие приложения с мобильного: лист выезжает с пружинной кривой iOS, фон (springboard) слегка отъезжает назад scale(.96) и темнеет — как настоящий app-переход iPhone.
- Landscape на телефоне не должен разваливаться (допустимо просто оставаться в portrait-раскладке, но без наложений).

## Порядок и правила

1. Фаза разведки (параллельно, Opus): (а) уточнить рыночные цены РФ для устройств B1 (WebSearch: готовые системники RTX 4090/5090, Mac Studio M4 Max, DNS/Ситилинк/регард) → файл `research/devices.md`; (б) собрать URL-ы и краткие описания кейсов для B3 (репо Shahen777/urartu-sites через gh api, сайты urartu.pro) → `research/portfolio.md`.
2. Лоты строго последовательно: A → B → C (общие файлы index.html/os.css/os.js/i18n.js).
3. После каждого лота: node --check обоих JS, curl 200, самопроверка скриншотами (глазами!), краткий отчёт.
4. Не ломать: магнификацию дока (52→84, косинус R=132, пружина 190/20), drag/resize окон, темы, i18n, звонок, автосоздание iframe агентов при первом открытии, JSON-LD (валидность), политику конфиденциальности.
5. Ноль внешних ресурсов при загрузке страницы. Новые изображения — локальные, оптимизированные (webp+png, суммарно портфолио ≤500KB).
6. Все новые строки — в ОБА словаря i18n. Русская типографика («ёлочки», тире).
7. Приёмка (Fable): полный регресс-чеклист + скриншоты всех режимов + замер веса страницы; починить найденное самостоятельно.
